# StatCheck 사용설명

## 1. StatCheck란?

`StatCheck`는 Unreal Editor 안에서 최적화 후보를 찾기 위한 버튼 기반 진단 플러그인입니다.

실시간 추적을 기본으로 하지 않고, 사용자가 버튼을 눌렀을 때만 FPS, DrawCall, Actor 수, Component 수, 선택 Actor 정보 등을 CSV 로그로 저장합니다.

## 2. 외부 Unreal 프로젝트에서 사용하는 방법

### 1단계: Unreal Editor 닫기

대상 프로젝트가 열려 있다면 먼저 Unreal Editor를 닫습니다.

플러그인 DLL이 사용 중이면 복사나 빌드가 실패할 수 있습니다.

### 2단계: 플러그인 폴더 복사

현재 프로젝트의 아래 폴더를 복사합니다.

```text
Plugins/StatCheck
```

대상 Unreal 프로젝트 안에 아래 위치로 붙여 넣습니다.

```text
대상프로젝트/Plugins/StatCheck
```

`Plugins` 폴더가 없다면 새로 만들면 됩니다.

### 3단계: 불필요한 빌드 산출물 정리

다른 프로젝트로 옮길 때는 아래 폴더가 있으면 삭제해도 됩니다.

```text
Plugins/StatCheck/Binaries
Plugins/StatCheck/Intermediate
```

이 폴더들은 대상 프로젝트에서 다시 빌드됩니다.

### 4단계: 프로젝트 열기

대상 프로젝트의 `.uproject`를 엽니다.

Unreal이 새 플러그인 모듈을 빌드하겠다고 물어보면 빌드를 진행합니다.

C++ 프로젝트라면 Visual Studio 또는 Rider에서 프로젝트 파일을 다시 생성한 뒤 빌드해도 됩니다.

### 5단계: 플러그인 활성화 확인

Unreal Editor에서 다음 메뉴를 확인합니다.

```text
Edit > Plugins
```

`StatCheck`가 활성화되어 있는지 확인합니다.

활성화 후 재시작을 요구하면 Editor를 재시작합니다.

## 3. StatCheck 창 열기

Unreal Editor 상단 메뉴에서 엽니다.

```text
Window > StatCheck
```

창 상단의 `?` 버튼을 누르면 Unreal 내부 팝업으로 간단한 사용법을 볼 수 있습니다.

## 4. 기본 버튼 설명

### Save Snapshot Log

현재 타이밍의 최적화 정보를 한 번 수집해서 CSV로 저장합니다.

주요 저장 값:

- FPS
- FrameTime
- GameThread 근사값
- DrawCall
- 메모리 사용량
- 현재 에디터 뷰포트 기준 보이는 Actor 수
- 전체 Actor 수
- Hidden Actor 수
- Tick 활성 Actor 수
- Primitive Component 수
- StaticMesh Component 수
- SkeletalMesh Component 수
- Light Component 수
- Material Slot 수
- StaticMesh 기준 대략 Triangle 수
- 선택 Actor 비용 proxy

### Record 10s Log

버튼을 누른 순간부터 10초 동안 1초 간격으로 Snapshot을 기록합니다.

실시간 추적 기능이 아니라, 사용자가 기록을 시작했을 때만 잠깐 샘플링합니다.

### Delete Logs

StatCheck가 저장한 CSV 로그를 삭제합니다.

삭제 대상 폴더:

```text
Saved/StatCheck/Logs
```

## 5. 리포트 버튼 설명

### Actor Tick Report

Tick이 켜진 Actor 목록을 CSV로 저장합니다.

최적화할 때 Tick을 꺼도 되는 Actor를 찾는 데 유용합니다.

### Component Report

현재 월드의 Component 타입별 개수를 CSV로 저장합니다.

StaticMesh, SkeletalMesh, Light, Collision 관련 Component가 어느 정도 있는지 확인할 수 있습니다.

### Selected Actor Report

현재 선택한 Actor의 Component 상세 정보를 CSV로 저장합니다.

선택 Actor가 가진 Component, Material Slot, Collision, Bounds, Triangle 후보 값을 확인할 수 있습니다.

### Top Actors Report

Component 수, Material Slot, Triangle 수, Tick 여부 등을 기준으로 무거운 Actor 후보 상위 50개를 CSV로 저장합니다.

이 값은 실제 GPU 비용을 완전히 뜻하지는 않고, 최적화 후보를 찾기 위한 proxy 값입니다.

### Set Baseline

현재 Snapshot을 비교 기준으로 저장합니다.

최적화 작업을 하기 전에 누르면 좋습니다.

### Compare Baseline

현재 Snapshot을 다시 수집한 뒤 `Set Baseline`으로 저장한 기준값과 비교합니다.

최적화 전후 차이를 CSV로 남길 수 있습니다.

## 6. 로그 저장 위치

모든 CSV 로그는 현재 Unreal 프로젝트 기준으로 아래 폴더에 저장됩니다.

```text
Saved/StatCheck/Logs
```

예시 파일 이름:

```text
StatCheckSnapshot_20260526-143000.csv
StatCheckProfile_20260526-143000.csv
StatCheckActorTickReport_20260526-143000.csv
StatCheckComponentReport_20260526-143000.csv
StatCheckSelectedActorReport_20260526-143000.csv
StatCheckTopActorsReport_20260526-143000.csv
StatCheckComparisonReport_20260526-143000.csv
```

## 7. 권장 사용 흐름

1. `Window > StatCheck`를 엽니다.
2. 최적화 전 `Save Snapshot Log`를 누릅니다.
3. 필요하면 `Set Baseline`을 누릅니다.
4. `Actor Tick Report`, `Component Report`, `Top Actors Report`로 후보를 찾습니다.
5. 특정 Actor가 의심되면 Actor를 선택하고 `Selected Actor Report`를 누릅니다.
6. 최적화 작업을 합니다.
7. `Compare Baseline`을 눌러 전후 차이를 기록합니다.

## 8. 주의사항

- StatCheck는 화면을 사진으로 찍어 저장하지 않습니다.
- CSV에는 숫자와 이름 정보만 저장합니다.
- `Visible Actor Count`는 에디터 뷰포트 기준의 추정값입니다. 실제 렌더러가 최종 draw한 객체 수와 완전히 같지는 않습니다.
- `Approx Triangle Count`는 StaticMesh 기준 대략값입니다. Nanite, LOD, Occlusion 결과를 완전히 반영하지 않습니다.
- 큰 맵에서 리포트 버튼을 누르면 순간적으로 Actor와 Component를 순회하므로 짧은 멈춤이 생길 수 있습니다.
